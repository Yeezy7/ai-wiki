# 如何使用

## 文章结构

每篇 CV Wiki 文章都遵循统一的结构，方便不同场景使用：

| 章节 | 用途 |
|------|------|
| 一句话解释 | 快速了解概念 |
| 它解决什么问题 | 理解背景和动机 |
| 核心思想 | 抓住本质 |
| 算法流程 | 理解步骤 |
| 数学定义 | 形式化理解 |
| 代码示例 | 动手实践 |
| 面试标准回答 | 面试准备 |
| 高频追问 | 深入理解 |
| 工程实践 | 实际应用 |
| 常见误区 | 避免踩坑 |
| 参考资料 | 延伸阅读 |

## 使用场景

### 场景 1：学习新概念

1. 先看「一句话解释」了解是什么
2. 再看「它解决什么问题」理解为什么需要
3. 看「核心思想」抓住本质
4. 看「代码示例」动手实践

### 场景 2：面试准备

1. 看「一句话解释」准备开场白
2. 看「面试标准回答」准备口述
3. 看「高频追问」准备深入回答
4. 看「常见误区」避免踩坑

### 场景 3：工程查阅

1. 看「代码示例」快速上手
2. 看「工程实践」了解实际注意事项
3. 看「常见误区」避免踩坑

## 文章状态

每篇文章都有状态标记：

- **draft** — 草稿，内容可能不完整
- **review** — 待审阅，内容基本完整
- **stable** — 稳定版本，经过验证
